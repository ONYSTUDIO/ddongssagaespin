# ADMIN_DASHBOARD.md

# 똥싸당 운세 - 관리자 대시보드 기획

## 목적

통계 페이지와 관리자 페이지를 하나의 통합 대시보드로 구성한다.

접근 주체는 개발자(운영자) 본인이며,
게임 데이터를 실시간으로 조회하고 서비스 운영에 필요한 관리 기능을 제공한다.

---

## 제작 이유

- AI 기반 1인 개발 학습
- DevOps 실무 능력 향상
  - 인증 레이어 설계
  - 환경 분리 (dev / prod)
  - 접근 제어
  - 실시간 데이터 조회

---

## 왜 하나로 합치는가

- 접근 주체가 동일 (개발자 본인)
- 통계와 어드민이 동일한 데이터 참조
- 두 개로 나누면 인증 / 배포 / 유지보수 비용이 두 배
- 현재 MVP 단계에서 분리는 오버엔지니어링

향후 통계 페이지를 외부에 공개할 경우 분리 검토.

---

## URL 구조

```
/admin
  /admin/stats        통계 (DAU, 스핀 수, 카드 통계)
  /admin/users        회원 조회 및 관리
  /admin/spin-logs    스핀 기록 조회
  /admin/fortune      포춘쿠키 메시지 관리
```

---

## 페이지 구성

### 1. 통계 (Stats)

서비스 현황을 한눈에 파악한다.

| 항목 | 내용 |
|------|------|
| DAU | 일간 활성 사용자 수 |
| 가입자 수 | 전체 누적 가입자 |
| 스핀 수 | 일별 / 주별 / 월별 |
| 카드 통계 | 등급별 획득 비율 (S/A/B/C/D) |
| 심볼 통계 | 슬롯 심볼 출현 횟수 및 확률 |

---

### 2. 회원 관리 (Users)

| 기능 | 내용 |
|------|------|
| 회원 목록 조회 | 가입일, 최근 접속일, 스핀 잔여 수 |
| 회원 검색 | 아이디 기반 |
| 스핀 수 조정 | 수동 지급 / 차감 |
| 계정 정지 | 어뷰징 대응 |

---

### 3. 스핀 기록 (Spin Logs)

| 기능 | 내용 |
|------|------|
| 전체 스핀 로그 | 날짜 / 유저 / 결과 / 점수 |
| 필터 | 날짜 범위 / 등급 / 유저 |
| 이상 탐지 | 단시간 대량 스핀 감지 |

---

### 4. 포춘쿠키 관리 (Fortune)

| 기능 | 내용 |
|------|------|
| 메시지 목록 | 현재 등록된 포춘쿠키 메시지 조회 |
| 메시지 추가 | 새 메시지 등록 |
| 메시지 수정 | 기존 메시지 편집 |
| 메시지 삭제 | 부적절한 메시지 제거 |

---

## 인증 설계

관리자 전용 인증을 별도로 구성한다.

### 방향

- Supabase Row Level Security (RLS) 활용
- 관리자 계정 테이블 별도 관리
- 또는 Cloudflare Access로 IP / 이메일 기반 접근 제한

### 인증 흐름

```
1. /admin 접근
2. 관리자 로그인 (일반 회원 로그인과 분리)
3. JWT 또는 세션 발급
4. 인증 성공 시 대시보드 진입
```

## 기술 스택

| 항목 | 내용 |
|------|------|
| Frontend | React + TypeScript + Vite (기존과 동일) |
| DB 조회 | Supabase (Realtime 활용 검토) |
| 접근 제어 | Cloudflare Access 또는 Supabase RLS |
| 배포 | Cloudflare Workers (기존 배포 환경 활용) |

---

## 개발 순서

1단계 선행 조건

- Supabase DB 테이블 완성 (users, spin_logs, rankings, fortune_cards)
- 회원가입 / 로그인 완성

2단계 어드민 구축

1. 관리자 인증 구현
2. 통계 페이지 (읽기 전용)
3. 회원 관리 페이지
4. 스핀 로그 조회
5. 포춘쿠키 메시지 관리

---

## DevOps 학습 포인트

이 대시보드를 만들면서 아래 내용을 직접 경험한다.

- Supabase RLS 정책 설계
- 환경 변수 분리 (.env.local / .env.production)
- Cloudflare Access를 통한 URL 단위 접근 제어
- Supabase Realtime으로 실시간 데이터 반영
- 관리자 전용 API 설계 및 권한 분리

---

## 주의 사항

- 어드민 페이지 URL은 외부에 노출하지 않는다
- 관리자 계정은 일반 회원 테이블과 분리한다
- 프로덕션 DB 직접 수정은 어드민 페이지를 통해서만 수행한다
- MVP 출시 전까지 어드민 기능은 읽기 전용으로만 구성한다
